A : Change AI script sets the script that you specify to take effect on the target creature at the level you specify. Override level means it overrides all other scripts. It doesn't mean that the other scripts are altered. They aren't the scripts at all other levels are still there it's just that if the override script tells the cre to do something at the same time as another script tells it to do some action, the action in the override script will be performed.

A : ChangeAIScript changes the script in the target permanantly.
Be careful about changing any creature's override script, I have noticed that many NPC's and PC's use override scripts to interact with the party, each other and the events around them.

A : Unfortunately, there is no way to check which AI script is set on the creature. And yes, setting a script means that any previoously set script is now ..not set hehe...or "blown-away" as you put it. 

The General, Default etc. are levels of script. So you can have more than one script running on a creature at once. One at the deafult level, one at the general level etc. THat is why I said what I did about the OVerride script taking precedence over the others...but not not deleting them from the creature because it is at a different level. To sum it up, any creature may have up to 5 different scripts running on it...in the default, general, race, class and override 'slots'. I think Specifics is for something else....

A : All the different script locations are in specific order based on weight. 

Heres the order:

0 OVERRIDE
1 AREA
2 SPECIFICS
4 CLASS
5 RACE
6 GENERAL
7 DEFAULT

That means that the parser starts at the O location and works it way downward until it runs through all seven locations. 

As you can see, specifics is indeed a script location, however this location can also hold a value, or if you add text to your SPECIFIC.ids, it can also hold text. 

However I would not modify the SPECIFIC location, on PC's like Edwin or Korgan, etc. As some of them run scripts from this location. 

Some creatures in game (such as Adalon) also use this location for specific NPC data, telling them to attack certain things, change this and you mess up those events. 

A : As to script levels, % are reflected in the cre files and savegame cre's.

0 OVERRIDE Character, always active
1 AREA Stored in the current AREA.
2 SPECIFICS Specifics is stored in the cre, to quote the original IESDP: 

0x0274 1 (byte) "Specific" type (see SPECIFIC.IDS).

I have never tried changing scripts at this level, however ( I always use ChangeSpecifics for not repeating actions on cres ), but as you say the game does, and I know others have as well. 

The games do no use Scroll level three...there is not such a location, as the IDS files simply cross reference text to engine internals. So if there was such a location that could be used, it would be in the IDS files. However, you can compile a script using that location (3) in the script, however I am almost positive it will not change anything (good to test this though, see what happens...I could be wrong =) ). 
4 CLASS Character, active when party scripts are active
5 RACE Character, active when party scripts are active
6 GENERAL Character, active when party scripts are active
7 DEFAULT Character, active when party scripts are active
