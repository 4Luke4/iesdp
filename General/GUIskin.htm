Interface Skinning in Infinity Engine games


By SeEd

This would be a good way to expose GUI Editing 

I’m sure you know about the so-called ”skins”. Programs like WinAmp and ICQ can be customized to look exactly like you want it to. Now, so can the Infinity Engine games…And it’s not very hard to do!

The GUI (Graphics User Interface) editing is a small part of todays IE editing. However, it is a way to really enhance your game and give it a unique look. In this tutorial, I will explain how to modify your Icewind Dale start screen, but let’s hear something about the basics first. If you already know about BAM and MOS files, you can skip the following section. 
In this tutorial, I assume you’re familiar with extracting BIF files and using the override folder.

Needed tools: Image editor, Bifftoool, BAMWorkshop and MOSWorkshop

Okay, extract the GUIbam.bif and GUImos.bif using the bifftool

NOTE: This tutorial can be used for any Infinity Engine game, but you must note that some file names might not be the same!

MOS and BAM files

You can split the GUI into two parts:

- The background
- The buttons


The background of the GUI is a single MOS file. MOS files are flat, static images. There can be NO animations in a MOS file. They are comparable to a BMP or JPEG image.

The buttons in the GUI are BAM files. BAM files are the main hub of graphics in the Infinity Engine: Spell, item and weapon icons, character animations, Spell animations, GUI buttons…All that (and more) is made of BAM files. The BAM files are split up in sequences, which contain the various frames of the animation. What we want to look at is the button BAMs. Each sequence is a button, and each frame is a bitmap picture. 

Here is how the sequences in the START.bam (and most other button BAMs) looks

Sequence 1, 2, 3, 4, 5
- Frame 1: This is the button´s normal state. 
- Frame 2: This is the button´s down state.
- Frame 3: This is the button’s greyed-out state.

The normal state is displayed as default. 
The down state is displayed when the button is clicked
The greyed out state is displayed when the buttons option is unavailable.

That's the scoop on BAMs, now, let's take a closer peek on the MOS files.


The background of your skin 

Open up the START.mos file in MOSWorkshop. This is the startup background from Icewind Dale, I’m sure you recognize it. Now, let’s edit it! Export the MOS file as a BMP or something similiar, and open it up in a graphics editing program. PhotoShop or Paint Shop Pro are both good. You can also make a new one from scratch, just make sure it is 640 pixels wide and 480 pixels high. This is not a PhotoShop or Paint Shop Pro tutorial, but I’ll give you some tips and tricks anyway:

- Remember the placement of the buttons. You could for example take a screenshot and use it for reference. It looks good if you made the buttons so they fit well in with the design of the MOS file. Look at how Black Isle did it: They made a small area of stone, where the buttons would fit perfectly in. 

- Remember to focus on the piece in the foreground

- Don’t make your image too bright or with too strong colors. This will look messy and somewhat childish.

- If you plan on editing more than one screen, apply the same design to all screens! It looks fucked up if you go from a very reddish screen with a medieval design to a bluish screen with the a futuristic design, and then to a greenish screen with an egyptic design!

Feel free to use the one in the Zip file, since it’s your first time.

Import your image into MOSWorkshop, and go to Save As, and save it as START.mos in your override folder. It is very important to get the name right, or the game will ignore the changes. Try starting your game. Whoa, eh? Looks kewl, doesn’t it? But the buttons seem kinda’ malplaced, so we’ll get down to changing those in just a minute. Now light a cigar and smile happily, knowing that you’ve taken the first step to a fully customized look of your game.


The buttons

Open up START.bam in BAMWorkshop. This is the button panel for the startup screen. There is 5 sequences, one for each button. You can make individual buttons, or just 5 of the same, but remember that the total number of sequenzes and frames must be exactly the same as before you edited it. If there’s just one flaw, it will look wrong. Remember that in Infinity Engine games, the turqoise color (R: 0, G: 151, B: 151) equals transparency, and the pink color (R: 255, G: 101, B: 151) equals shadow.

Again, feel free to do use the ones in the Zip. Here’s how to get it right:
Save the three JPGs in the Zip named: STARTnormal.jpg, STARTdown.jpg and STARTgrey.jpg 

(or make your own – it looks best if they're 170 x 46), and open up BAMWorkshop. Go to Edit | Sequence | Add Sequence, and you will see taht Sequence 0 has been added. So far, so good. Now go to the ”Frames” tab in the topright corner of BAMWorkshop. Go to Edit | Frame | Import Frame, and find the files you want to use as buttons. First, you import the normal state button, then the down state button and then the greyed out state button. Now you have a full sequence. Repeat this 5 times.
